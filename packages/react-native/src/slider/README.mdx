# Slider

**Category**: input | **Tags**: form, range, adjustment

The Slider component provides a range control for selecting numeric values within a defined minimum and maximum. It supports both single value selection with one thumb and range selection with two thumbs, making it ideal for settings, filters, and any scenario requiring intuitive value adjustment.

## Props

### Value Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| value | number \| [number, number] | required | Current value or range |
| onChange | (value: number \| [number, number]) => void | required | Value change handler |
| onBlur | () => void | undefined | Called when touch is released |

### Constraint Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| min | number | 0 | Minimum allowed value |
| max | number | 100 | Maximum allowed value |
| step | number | 1 | Value increment (0 for continuous) |

### Behavior Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| size | "sm" \| "md" \| "lg" | "md" | Size preset for thumb |
| disabled | boolean | false | Disables interaction |
| showValue | boolean | false | Shows value above thumb(s) |

### Styling Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| color | string | undefined | Custom active track color |
| style | StyleProp\<ViewStyle\> | undefined | Custom style overrides |

## Accessibility

**WCAG Level:** AA compliant

**Touch Target:**

- Minimum touch target follows thumb size
- Entire track is interactive for easier targeting

**Screen Reader:**

- Uses semantic slider role when available
- Supports custom accessibilityLabel
- Value changes are announced

**Visual Requirements:**

- Disabled state indicated by reduced opacity (0.5)
- Active track provides clear visual feedback
- Thumb has shadow for depth perception
