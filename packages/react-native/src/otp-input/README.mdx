# OTPInput

One-time password input with multiple fields for verification codes received via email or SMS. Features auto-advance between fields, paste support, and auto-submit on completion.

## Import

```tsx
import { OTPInput } from "@repo/react-native";
```

## Usage

### Basic

<View style={{ padding: 20 }}>
  <OTPInput value="" onChange={() => {}} />
</View>

```tsx
<OTPInput
  value={otp}
  onChange={setOtp}
  onComplete={(code) => submitOTP(code)}
/>
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | - | Current OTP value (controlled) |
| `onChange` | `(otp: string) => void` | - | Callback when OTP changes |
| `onComplete` | `(otp: string) => void` | `undefined` | Callback when all fields filled |
| `length` | `number` | `6` | Number of OTP digits |
| `type` | `"number" \| "text"` | `"number"` | Input type (affects keyboard) |
| `size` | `"sm" \| "md" \| "lg"` | `"md"` | Size preset for fields |
| `variant` | `"outline" \| "filled"` | `"outline"` | Visual style of fields |
| `error` | `boolean` | `false` | Error state (red borders) |
| `disabled` | `boolean` | `false` | Disables all fields |
| `autoFocus` | `boolean` | `true` | Auto-focus first field on mount |
| `style` | `StyleProp<ViewStyle>` | `undefined` | Container custom styles |
| `accessibilityLabel` | `string` | `undefined` | Screen reader label |

## Accessibility

- Container has `accessibilityLabel` for screen readers
- Each field labeled as "Digit N of M"
- Full keyboard navigation support
- Screen reader announces field changes
- WCAG AA compliant color contrast

## Behavior

- **Auto-advance**: Moves to next field on input
- **Backspace**: Moves to previous field when current is empty
- **Paste**: Distributes OTP across fields automatically
- **Auto-submit**: Calls `onComplete` and dismisses keyboard when filled
- **Success state**: Green borders when all fields filled

## Best Practices

1. Always handle `onComplete` callback for submission
2. Clear OTP value on error to allow re-entry
3. Use `type="number"` (default) for OTP codes
4. Provide visual feedback via `error` prop for invalid codes
