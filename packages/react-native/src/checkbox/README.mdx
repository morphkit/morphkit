The Checkbox component provides a selectable control for boolean inputs with support for checked, unchecked, and indeterminate states. It enables multi-selection and mixed-state scenarios in forms and lists, following platform conventions for touch interaction and accessibility.

## Props

### State Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| checked | boolean | required | Current checked state (controlled) |
| indeterminate | boolean | false | Shows minus icon for partial selection |
| disabled | boolean | false | Disables interaction and applies disabled styling |

### Event Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| onChange | (checked: boolean) => void | required | Called when checkbox is toggled |
| onBlur | () => void | undefined | Called when press is released |

### Appearance Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| size | "sm" \| "md" \| "lg" | "md" | Size preset for checkbox box |
| color | string | undefined | Custom color override for checked state |

### Content Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| children | ReactNode | undefined | Label content (typically Typography) |
| name | string | undefined | Form field name for identification |

### Styling Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| style | StyleProp\<ViewStyle\> | undefined | Custom style overrides |

## Accessibility

**WCAG Level:** AA compliant

**Screen Reader:**

- Announces role as "checkbox"
- Announces checked state (checked, unchecked)
- Announces disabled state when disabled
- Custom labels supported via accessibilityLabel prop

**Touch Target:**

- Minimum touch target: 48pt x 48pt (Apple HIG)
- Touch area includes both box and label

**Visual Requirements:**

- Color contrast meets 4.5:1 for checkmark icon
- Disabled state indicated by opacity (0.5)
- Border provides clear boundary in unchecked state
- Indeterminate state visually distinct with minus icon
