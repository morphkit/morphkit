# Typography Component

A comprehensive text component with 11 iOS-inspired typography variants using React Native's StyleSheet API.

## Features

- 11 semantic text variants following iOS Human Interface Guidelines
- Built with React Native StyleSheet for optimal performance
- Type-safe variant API with autocomplete
- Style prop support for custom styling
- Accessible by default with React Native Text props

## Basic Usage

The Typography component defaults to the "body" variant:

<View style={{ padding: 16, backgroundColor: '#f9fafb', borderRadius: 8, marginVertical: 12 }}>
  <Typography>This is body text with default styling</Typography>
</View>

## Variant Overview

| Variant | Size | Weight | Use Case |
|---------|------|--------|----------|
| large-title | 34pt | Regular | Top-level navigation, splash screens |
| title-1 | 28pt | Regular | Primary page titles |
| title-2 | 22pt | Regular | Section headers |
| title-3 | 20pt | Regular | Subsection headers |
| heading | 17pt | Semibold | Emphasized headings |
| body | 17pt | Regular | Main content, paragraphs |
| callout | 16pt | Regular | Highlighted information |
| subhead | 15pt | Regular | List headings, secondary content |
| footnote | 13pt | Regular | Explanatory notes, metadata |
| caption-1 | 12pt | Regular | Image captions, labels |
| caption-2 | 11pt | Regular | Tertiary information |

## Custom Styling

Combine variants with style prop:

<View style={{ padding: 16, backgroundColor: '#f9fafb', borderRadius: 8, marginVertical: 12 }}>
  <Typography variant="title-1" style={{ color: '#3b82f6' }}>
    Colored Title
  </Typography>
  <Typography variant="body" style={{ color: '#6b7280', fontStyle: 'italic', marginTop: 8 }}>
    Italic gray body text
  </Typography>
  <Typography variant="caption-1" style={{ color: '#ef4444', textTransform: 'uppercase', marginTop: 8 }}>
    Uppercase caption
  </Typography>
</View>

## Accessibility

All variants include proper accessibility support:

<View style={{ padding: 16, backgroundColor: '#f9fafb', borderRadius: 8, marginVertical: 12 }}>
  <Typography
    variant="title-1"
    accessibilityRole="header"
    accessibilityLabel="Main page heading"
  >
    Accessible Heading
  </Typography>
  <Typography
    variant="body"
    accessibilityHint="Tap to read more details"
    style={{ marginTop: 8 }}
  >
    This content has accessibility hints for screen readers
  </Typography>
</View>

## Props

| Prop | Type | Description | Default |
|------|------|-------------|------------|
| variant | TypographyVariant | Text style variant | "body" |
| children | ReactNode | Content to render | - |
| style | `StyleProp<TextStyle>` | Custom styles to apply | - |
| ...props | TextProps | All React Native Text props | - |

## Best Practices

1. **Use semantic variants**: Choose variants based on content hierarchy, not just visual appearance
2. **Maintain consistency**: Use the same variants for similar content types across your app
3. **Respect accessibility**: Always provide appropriate `accessibilityRole` and `accessibilityLabel` props
4. **Test across platforms**: Verify typography works well on both iOS and Android
5. **Style prop for customization**: Use the style prop to override specific properties when needed
