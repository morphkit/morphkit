# Radio

Single-selection radio button control with group management for choosing one option from a set. Ensures only one option can be selected at a time within a group with clear visual indication of selection state.

## Import

```tsx
import { RadioGroup, RadioButton } from "@repo/react-native";
```

## Usage

### Basic

<View style={{ padding: 20 }}>
  <RadioGroup value="option1" onValueChange={() => {}}>
    <Stack gap={12}>
      <RadioButton value="option1">
        <Typography variant="body">Option 1</Typography>
      </RadioButton>
      <RadioButton value="option2">
        <Typography variant="body">Option 2</Typography>
      </RadioButton>
      <RadioButton value="option3">
        <Typography variant="body">Option 3</Typography>
      </RadioButton>
    </Stack>
  </RadioGroup>
</View>

```tsx
<RadioGroup value={selected} onValueChange={setSelected}>
  <RadioButton value="option1">
    <Typography variant="body">Option 1</Typography>
  </RadioButton>
  <RadioButton value="option2">
    <Typography variant="body">Option 2</Typography>
  </RadioButton>
  <RadioButton value="option3">
    <Typography variant="body">Option 3</Typography>
  </RadioButton>
</RadioGroup>
```

      <RadioButton value="5" accessibilityLabel="Rating 5" />

## Props

### RadioGroup Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | - | Currently selected RadioButton value (controlled) |
| `onValueChange` | `(value: string) => void` | - | Callback when selection changes |
| `children` | `ReactNode` | - | RadioButton components |
| `disabled` | `boolean` | `false` | Disables all radio buttons in group |
| `style` | `StyleProp<ViewStyle>` | `undefined` | Additional custom styles for group container |

The component also accepts all standard React Native `ViewProps`.

### RadioButton Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | - | Unique identifier for this radio button option |
| `children` | `ReactNode` | `undefined` | Content displayed next to radio button (typically Typography) |
| `disabled` | `boolean` | `false` | Disables this specific radio button |
| `size` | `"sm" \| "md" \| "lg"` | `"md"` | Size of radio circle |
| `style` | `StyleProp<ViewStyle>` | `undefined` | Additional custom styles for button wrapper |

The component also accepts all standard React Native `ViewProps`.

## Accessibility

The Radio components follow accessibility best practices:

- **RadioGroup**: Uses `accessibilityRole="radiogroup"` for proper grouping
- **RadioButton**: Uses `accessibilityRole="radio"` for screen reader identification
- Provides `accessibilityState` with `checked` and `disabled` states
- Supports custom `accessibilityLabel` prop for screen readers
- Maintains minimum 44x44 point touch target
- Keyboard support on web (Arrow keys navigate, Space selects)

## Theme Support

The component automatically adapts to light and dark modes:

- **Light mode**: Gray border (#9CA3AF) when unselected, primary blue (#4A90E2) when selected
- **Dark mode**: Lighter gray border (#6B7280) when unselected, lighter blue (#5AA2F5) when selected

## Best Practices

1. **Single Selection**: Use radio buttons when only one option can be selected
2. **Labels**: Always provide clear labels for better UX
3. **Default Selection**: Consider setting a default selected value
4. **Grouping**: Use Stack component for consistent spacing between options
5. **Multiple Choice**: Use Checkbox component instead if multiple selections are allowed
