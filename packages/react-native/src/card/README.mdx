# Card

**Category**: surfaces | **Tags**: container, content, elevation

The Card component provides a content container with elevation, borders, and padding for grouping related information into visually distinct sections. It supports multiple visual variants for different hierarchy levels and optional press behavior for interactive cards.

## Props

### Behavior Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| variant | "elevated" \| "outline" \| "ghost" \| "filled" | "elevated" | Visual style variant |
| size | "sm" \| "md" \| "lg" | "md" | Padding size preset |
| onPress | () => void | undefined | Press handler, enables pressable behavior |

### Content Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| children | ReactNode | undefined | Card content |

### Styling Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |

### ViewProps

Card extends ViewProps and forwards all additional props to the root element.

## Accessibility

**WCAG Level:** AA compliant

**Screen Reader:**

- Announces role as "button" when onPress is provided
- Static cards do not announce button role
- Content is accessible to screen readers

**Touch Target:**

- Pressable cards respond to touch input
- Visual feedback via opacity change on press

**Visual Requirements:**

- Color contrast meets 4.5:1 for text content
- Press feedback visible via opacity change (0.8)
- Elevation provides visual hierarchy through shadows
