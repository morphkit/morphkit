# Switch

Toggle switch for binary on/off states with iOS and Android platform-specific styling. Provides familiar toggle interaction for boolean settings with clear visual indication of current state.

## Import

```tsx
import { Switch } from "@repo/react-native";
```

## Usage

### Basic

<View style={{ padding: 20 }}>
  <Switch
    checked={false}
    onCheckedChange={() => {}}
    label="Enable Notifications"
  />
</View>

```tsx
<Switch
  checked={notificationsEnabled}
  onCheckedChange={setNotificationsEnabled}
  label="Enable Notifications"
/>
```

## Props

### Switch Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `checked` | `boolean` | - | Current on/off state (controlled) |
| `onCheckedChange` | `(checked: boolean) => void` | - | Callback when user toggles switch |
| `label` | `string` | `undefined` | Text label displayed next to switch |
| `disabled` | `boolean` | `false` | Disables switch interaction |
| `size` | `"sm" \| "md" \| "lg"` | `"md"` | Size of switch track and thumb |
| `color` | `string` | theme primary | Color of track when checked |
| `style` | `StyleProp<ViewStyle>` | `undefined` | Additional custom styles for wrapper |

The component also accepts all standard React Native `ViewProps`.

## Accessibility

The Switch component follows accessibility best practices:

- Uses `accessibilityRole="switch"` for proper screen reader identification
- Provides `accessibilityState` with `checked` status
- Supports `accessibilityLabel` from the `label` prop or custom prop
- Maintains minimum 44x44 point touch target
- Keyboard support on web (Space to toggle)
- Announces state changes to screen readers

## Theme Support

The component automatically adapts to light and dark modes:

- **Light mode**: Gray track (#D1D5DB) when off, primary blue (#4A90E2) when on
- **Dark mode**: Darker gray track (#6B7280) when off, lighter blue (#5AA2F5) when on
- White thumb in both themes
- Override with the `color` prop for custom branding

## Animation

The Switch component uses React Native's Animated API for smooth transitions:

- Thumb slides smoothly between on and off positions
- 200ms animation duration
- Uses native driver for optimal performance (60fps)

## Best Practices

1. **Binary Choices**: Use switches for on/off states, not multiple options
2. **Immediate Effect**: Switches should take effect immediately without requiring a save action
3. **Labels**: Provide clear labels that describe what the switch controls
4. **Settings**: Ideal for settings and preferences screens
5. **State Clarity**: The label should make it clear what "on" means
