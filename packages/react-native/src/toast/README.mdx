The Toast component provides temporary notification messages with slide-in animation. It supports 4 semantic variants for different message types, top or bottom positioning, and configurable auto-dismiss behavior for non-intrusive user feedback.

- **Info**: General information and neutral updates (default)
- **Success**: Confirmation of completed actions
- **Warning**: Caution messages requiring attention
- **Error**: Error states and failure notifications

## Props

### Required Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| visible | boolean | required | Controls toast visibility |
| message | string | required | Notification text content |

### Behavior Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| variant | "info" \| "success" \| "warning" \| "error" | "info" | Semantic visual style |
| position | "top" \| "bottom" | "bottom" | Screen placement |
| duration | number | 3000 | Auto-dismiss delay in ms (0 for manual) |
| onDismiss | () => void | undefined | Callback when toast should hide |

### Styling Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| style | StyleProp\<ViewStyle\> | undefined | Custom style overrides |

## Accessibility

**WCAG Level:** AA compliant

**Screen Reader:**

- Role set to "alert" for immediate announcement
- Live region set to "polite" to avoid interrupting current task
- Message content read automatically when visible

**Visual Requirements:**

- Color contrast meets 4.5:1 for text on all variant backgrounds
- Icon provides additional semantic context alongside color
- Shadow provides elevation distinction from content below
- Animation duration respects reduced motion preferences

**Best Practices:**

- Keep messages brief and actionable
- Use appropriate variants for message semantics
- Ensure sufficient duration for users to read content
- Provide manual dismiss option for important messages
