The Alert component displays inline notifications with semantic variants to communicate important information, status updates, or warnings to users. It supports four semantic variants (info, success, warning, error), optional dismiss functionality, and customizable icons.

## Basic Usage

## Variants

All available visual styles for different contexts.

<VariantsExample />

- **Info**: General informational messages, uses blue color scheme
- **Success**: Positive confirmations and completed actions, uses green color scheme
- **Warning**: Cautionary messages requiring attention, uses amber color scheme
- **Error**: Critical errors and failures, uses red color scheme

## Props

### Behavior Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| variant | "info" \| "success" \| "warning" \| "error" | "info" | Semantic variant for visual style |
| dismissible | boolean | false | Shows dismiss button when true |
| onDismiss | () => void | undefined | Callback when dismiss button pressed |

### Content Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| title | string | required | Primary alert message |
| description | string | undefined | Secondary supporting text |
| icon | ReactNode | undefined | Custom icon override |

### Styling Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| style | StyleProp\<ViewStyle\> | undefined | Custom style overrides |

## Accessibility

**WCAG Level:** AA compliant

**Screen Reader:**

- Container has accessibilityRole="alert"
- Screen readers announce content immediately as alert
- Dismiss button has accessibilityRole="button"
- Dismiss button has accessibilityLabel="Dismiss alert"

**Visual Requirements:**

- Color contrast meets 4.5:1 for text on variant backgrounds
- Icons use semantic colors for visual reinforcement
- Dismiss button has extended hitSlop for easier touch targeting
