The Input component provides a single-line text input field for collecting user text input in forms and interfaces. It supports labels, error states with validation messages, prefix/suffix icons, multiple input types (text, email, password, number), and two visual variants with built-in accessibility.

## Props

### Behavior Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| variant | "outline" \| "filled" | "outline" | Visual style variant |
| size | "sm" \| "md" \| "lg" | "md" | Size preset |
| type | "text" \| "email" \| "password" \| "number" | "text" | Input type affecting keyboard |
| disabled | boolean | false | Disables editing and applies disabled styling |

### Value Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| value | string | required | Current input value (controlled) |
| onChange | (text: string) => void | required | Callback when text changes |
| onBlur | () => void | undefined | Callback when input loses focus |
| placeholder | string | undefined | Placeholder text when empty |

### Label and Error Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| label | string | undefined | Label text displayed above input |
| name | string | undefined | Input name for form identification |
| error | string | undefined | Error message displayed below input |

### Icon Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| prefixIcon | ReactNode | undefined | Icon rendered before input field |
| suffixIcon | ReactNode | undefined | Icon rendered after input field |

### Styling Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| style | StyleProp\<ViewStyle\> | undefined | Custom style overrides for wrapper |

## Accessibility

**WCAG Level:** AA compliant

**Screen Reader:**

- accessibilityLabel is set from label prop, falling back to name prop
- Error messages announced via accessibilityLiveRegion="polite"
- Proper labeling ensures screen readers announce input purpose

**Keyboard Navigation:**

- Supports all standard TextInput keyboard behaviors
- Type-specific keyboards (email-address, numeric) for appropriate input types
- Secure text entry for password type

**Touch Target:**

- Minimum touch target: 36pt (sm), 44pt (md), 52pt (lg)
- Meets Apple HIG minimum of 44pt at default size

**Visual Requirements:**

- Color contrast meets 4.5:1 for text
- Error state clearly indicated by red border and error message
- Focus state indicated by border color change
- Disabled state indicated by reduced opacity (0.5)
