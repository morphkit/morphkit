# Input

Single-line text input field with label, placeholder, error state, and prefix/suffix icon support for various data entry needs. Provides clear visual states for focus, error, and disabled conditions with full accessibility support.

## Import

```tsx
import { Input } from "@repo/react-native";
```

## Usage

### Basic

<View style={{ padding: 20 }}>
  <Input
    label="Email"
    value=""
    onChangeText={() => {}}
    placeholder="you@example.com"
  />
</View>

```tsx
<Input
  label="Email"
  value={email}
  onChangeText={setEmail}
  placeholder="you@example.com"
  type="email"
/>
```

## Props

### Input Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | - | Current input value (controlled) |
| `onChangeText` | `(text: string) => void` | - | Callback when text changes |
| `label` | `string` | `undefined` | Text label above input |
| `placeholder` | `string` | `undefined` | Placeholder text shown when empty |
| `error` | `string` | `undefined` | Error message text shown below input |
| `disabled` | `boolean` | `false` | Disables input interaction |
| `prefixIcon` | `ReactNode` | `undefined` | Icon or element shown before text |
| `suffixIcon` | `ReactNode` | `undefined` | Icon or element shown after text |
| `size` | `"sm" \| "md" \| "lg"` | `"md"` | Size preset for height, fontSize, padding |
| `variant` | `"outline" \| "filled"` | `"outline"` | Visual style of input container |
| `type` | `"text" \| "email" \| "password" \| "number"` | `"text"` | Input type affecting keyboard and behavior |
| `style` | `StyleProp<ViewStyle>` | `undefined` | Additional custom styles for wrapper |

The component also accepts all standard React Native `TextInputProps` (except `style`).

## Accessibility

The Input component follows accessibility best practices:

- Uses `accessibilityLabel` from the `label` prop
- Sets `accessibilityInvalid` when an error is present
- Error text has `accessibilityLiveRegion="polite"` for announcements
- Supports keyboard navigation and focus management
- Maintains sufficient color contrast (WCAG AA compliant)

## Theme Support

The component automatically adapts to light and dark modes:

### Outline Variant
- **Light mode**: White background, gray border (#E5E7EB)
- **Dark mode**: Dark background (#374151), lighter border (#4B5563)

### Filled Variant
- **Light mode**: Light gray background (#F3F4F6), no border
- **Dark mode**: Lighter dark background (#4B5563), no border

### Focus & Error States
- **Focus**: Primary blue border (#4A90E2 light, #5AA2F5 dark)
- **Error**: Red border and text (#EF4444 light, #F87171 dark)

## Best Practices

1. **Labels**: Always provide a `label` for better UX and accessibility
2. **Placeholders**: Use placeholders as examples, not as labels
3. **Error Messages**: Provide clear, actionable error messages
4. **Input Types**: Use appropriate `type` prop for better keyboard experience
5. **Validation**: Show errors only after user interaction or form submission
