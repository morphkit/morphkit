# Select

Flexible inline selection component for choosing one or more options from visible choices. Provides a composable API where you control the visual presentation of options through custom child components.

## Import

```tsx
import { Select, SelectOption } from "@repo/react-native";
```

## Usage

### Basic Single Selection

<View style={{ padding: 20 }}>
  <Select value="opt1" onValueChange={() => {}}>
    <Stack gap={12}>
      <SelectOption value="opt1">
        <Card>
          <Typography variant="body">Option 1</Typography>
        </Card>
      </SelectOption>
      <SelectOption value="opt2">
        <Card>
          <Typography variant="body">Option 2</Typography>
        </Card>
      </SelectOption>
      <SelectOption value="opt3">
        <Card>
          <Typography variant="body">Option 3</Typography>
        </Card>
      </SelectOption>
    </Stack>
  </Select>
</View>

```tsx
const [value, setValue] = useState("");
```

### With Visual Selection State

**Important:** SelectOption does not apply selection styling. Your child components must handle visual feedback by checking the selection state.

<View style={{ padding: 20 }}>
  <Select value="tier-pro" onValueChange={() => {}}>
    <Stack gap={16}>
      <SelectOption value="tier-free">
        <Card
          variant={"tier-free" === "tier-pro" ? "elevated" : "default"}
          style={{ borderWidth: "tier-free" === "tier-pro" ? 2 : 1 }}
        >
          <Typography variant="heading">Free Tier</Typography>
          <Typography variant="body">Basic features</Typography>
        </Card>
      </SelectOption>
      <SelectOption value="tier-pro">
        <Card
          variant="elevated"
          style={{ borderWidth: 2 }}
        >
          <Typography variant="heading">Pro Tier</Typography>
          <Typography variant="body">All features</Typography>
        </Card>

    </Stack>
  </Select>

### Multiple Selection with Visual State

Show which items are selected in multi-select mode.

```tsx
const [contacts, setContacts] = useState<string[]>([]);

<Select value={contacts} onValueChange={setContacts} multiple>
  <Stack gap={12}>
    {allContacts.map(contact => (
      <SelectOption key={contact.id} value={contact.id}>
        <Card
          variant={contacts.includes(contact.id) ? "elevated" : "default"}
          style={{
            borderWidth: contacts.includes(contact.id) ? 2 : 1,
            borderColor: contacts.includes(contact.id) ? "#4A90E2" : "#E5E7EB"
          }}
        >
          <View style={{ flexDirection: "row", alignItems: "center", gap: 12 }}>
            <Typography variant="body">{contact.name}</Typography>
            {contacts.includes(contact.id) && (
              <Text style={{ color: "#4A90E2", fontSize: 18 }}>✓</Text>
            )}
          </View>
        </Card>
      </SelectOption>
    ))}
  </Stack>
</Select>
```

### Dynamic Options from Data

Map over data to create options dynamically.

```tsx
const tiers = [
  { id: "free", name: "Free Tier", price: "$0/mo", features: "Basic features" },
  { id: "pro", name: "Pro Tier", price: "$29/mo", features: "All features" },
  { id: "enterprise", name: "Enterprise", price: "Custom", features: "Custom solutions" }
];

const [selectedTier, setSelectedTier] = useState("pro");

<Select value={selectedTier} onValueChange={setSelectedTier}>
  <Stack gap={16}>
    {tiers.map(tier => (
      <SelectOption key={tier.id} value={tier.id}>
        <Card
          variant={selectedTier === tier.id ? "elevated" : "default"}
          style={{
            borderWidth: selectedTier === tier.id ? 2 : 1,
            borderColor: selectedTier === tier.id ? "#4A90E2" : "#E5E7EB"
          }}
        >
          <View>
            <Typography variant="heading">{tier.name}</Typography>
            <Typography variant="body">{tier.price}</Typography>
            <Typography variant="caption-1">{tier.features}</Typography>
          </View>
        </Card>
      </SelectOption>
    ))}
  </Stack>
</Select>
```

## Props

### Select Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string \| string[]` | - | Current selection (string for single mode, array for multiple mode) |
| `onValueChange` | `(value: string \| string[]) => void` | - | Callback when selection changes |
| `children` | `ReactNode` | - | SelectOption components and layout wrappers (e.g., Stack) |
| `multiple` | `boolean` | `false` | Enable multiple selection mode |
| `disabled` | `boolean` | `false` | Disable all options |
| `style` | `StyleProp<ViewStyle>` | `undefined` | Custom styles for container View |

The component also accepts all standard React Native `ViewProps`.

### SelectOption Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | - | Unique identifier for this option |
| `children` | `ReactNode` | - | Custom content to display (typically Card, View, or custom component) |
| `disabled` | `boolean` | `false` | Disable this specific option |
| `style` | `StyleProp<ViewStyle>` | `undefined` | Custom styles for Pressable wrapper |

The component also accepts all standard React Native `ViewProps`.

## Accessibility

The Select components follow accessibility best practices:

- **Select**: Uses `accessibilityRole="radiogroup"` for proper container identification
- **SelectOption**: Uses `accessibilityRole="radio"` for screen reader identification
- Provides `accessibilityState` with `checked` and `disabled` states
- Selection state changes are automatically announced to screen readers
- Supports custom `accessibilityLabel` props for enhanced accessibility

## Best Practices

1. **Visual Feedback**: Always implement visual selection feedback in your child components
   - Use border colors, background colors, or checkmarks to indicate selected state
   - Ensure sufficient contrast for accessibility

2. **Unique Values**: Provide unique `value` props for each SelectOption
   - Values are used to track selection state
   - Duplicate values will cause unexpected behavior

3. **Layout Components**: Use Stack or View to organize options
   - Stack provides consistent spacing between options
   - Allows for responsive grid layouts if needed

4. **Selection State**: Check selection state in child components
   - Single mode: `value === optionValue`
   - Multiple mode: `value.includes(optionValue)`

5. **Touch Targets**: Ensure options have sufficient size for touch interaction
   - Minimum 44x44 points for accessibility
   - Card components typically meet this requirement

6. **Performance**: For long lists, consider virtualization
   - FlatList can be used with SelectOption for better performance
   - Map operations work well for moderate lists (< 50 items)

## Common Patterns

### Tier Selection (Single)

```tsx
<Select value={tier} onValueChange={setTier}>
  <Stack gap={16}>
    {tiers.map(t => (
      <SelectOption key={t.id} value={t.id}>
        <Card variant={tier === t.id ? "elevated" : "default"}>
          <Typography variant="heading">{t.name}</Typography>
          <Typography variant="body">{t.description}</Typography>
        </Card>
      </SelectOption>
    ))}
  </Stack>
</Select>
```

### Contact Selection (Multiple)

```tsx
<Select value={selectedContacts} onValueChange={setSelectedContacts} multiple>
  <Stack gap={12}>
    {contacts.map(contact => (
      <SelectOption key={contact.id} value={contact.id}>

          <View style={{ flexDirection: "row", gap: 12 }}>
            <Typography>{contact.name}</Typography>
            {selectedContacts.includes(contact.id) && <Text>✓</Text>}
          </View>
        </Card>
      </SelectOption>
    ))}
  </Stack>
</Select>
```

## Important Notes

- **SelectOption does not apply selection styling** - you must implement visual feedback in your child components
- **SelectOption can be nested anywhere** in the children tree - the Context API works at any depth
- **Value prop type** - use `string` for single mode and `string[]` for multiple mode for proper type safety
- **Empty values** - empty string `""` or empty array `[]` are valid and mean nothing is selected
