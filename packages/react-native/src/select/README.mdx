The Select component provides a flexible inline selection interface using a compound component pattern. It supports both single and multiple selection modes with full accessibility, allowing custom visual implementations through SelectOption children. Unlike native pickers, it enables rich custom styling while maintaining proper ARIA semantics.

## Props

### Select Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| value | string \| string[] | required | Currently selected value(s) |
| onChange | (value: string \| string[]) => void | required | Handler called when selection changes |
| onBlur | () => void | undefined | Handler called when press completes |
| multiple | boolean | false | Enable multiple selection mode |
| disabled | boolean | false | Disable all options |
| children | ReactNode | required | SelectOption children |
| style | StyleProp\<ViewStyle\> | undefined | Custom container styles |

### SelectOption Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| value | string | required | Unique value for this option |
| onBlur | () => void | undefined | Handler called when press completes |
| disabled | boolean | false | Disable this specific option |
| children | ReactNode | required | Option content |
| style | StyleProp\<ViewStyle\> | undefined | Custom option styles |

## Accessibility

**WCAG Level:** AA compliant

**Semantic Roles:**

- Select container uses accessibilityRole="radiogroup"
- SelectOption uses accessibilityRole="radio"

**Screen Reader:**

- Announces checked state via accessibilityState.checked
- Announces disabled state via accessibilityState.disabled
- Custom labels supported via accessibilityLabel prop

**Implementation Notes:**

- The SelectOption component does not apply visual selection styling automatically
- Implement visual feedback in child components using the selection state
- Use theme tokens for consistent selection indicators (e.g., border color changes)

**Touch Target:**

- Ensure SelectOption children meet minimum 44pt touch target (Apple HIG)
- Use Card or similar components with adequate padding
